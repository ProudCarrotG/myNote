# 3.1并发进程
1. 顺序程序
	1. 顺序性：一条条语句顺序执行
	2. 封闭性：A 执行时所有资源都给 A 服务
	3. 确定性
	4. 可再现性
2. 进程关系：
	1. 无关
	2. 相关（临界资源）
		1. 竞争（互斥）：没有先后顺序
		2. 协作（同步）：存在先后顺序
	3. 判断方式 [[Bernstein条件]]
3. 错误原因：
	1. 对[临界区](临界区.md)的使用不当
4. 仅控制对相同[临界资源](临界资源.md)占用的进程
